## üîÑ Document Update Policy

### When to Update Documentation

1. **Requirements Changes**: Update `REQUIREMENTS.md` and corresponding spec files
2. **Design Changes**: Update `DESIGN.md` and enhancement suite `design.md`
3. **Implementation Changes**: Update `IMPLEMENTATION_PLAN.md` and `tasks.md`
4. **Architecture Changes**: Update `ARCHITECTURE.md`
5. **New Features**: Update relevant category documents

### Update Process

1. Identify affected documents
2. Update content with clear change descriptions
3. Update version/date in document header
4. Commit with descriptive message
5. Update this index if new documents are added
